EXPERIMENT 51 — Spectral Edit-Direction Cache Probe

Thread: fast-edit (Fast structure-preserving editing) · Status: done · Verdict: STRONG GO

A SeaCache-style idea adapted to editing. Instead of caching the full denoising prediction, cache the edit direction Δ_edit(t) = v_edit(t) − v_src(t). On 24 PIE-Bench edits with an instrumented FLUX.1-dev img2img denoiser, Δ_edit is 2.4× smoother than v_edit (smoother on 96% of steps) and caching it preserves quality ~13× better than full-prediction caching at equal step-skip. Delta caching dominates the entire speed–quality frontier. GO for a video follow-up.

Why

Diffusion editing cost scales with transformer evaluations. SeaCache-style caches skip recomputation when a low-pass-filtered state is temporally stable. We push this to editing: the part of the prediction that encodes what the edit does is the difference between the target- and source-prompt predictions. If that difference is smoother/lower-frequency than the full prediction, the edit direction is the right object to cache — freeze the edit, let the cheap base trajectory carry the rest.

Setup

FLUX.1-dev img2img (SDEdit), 4-bit NF4, 512px, 24 steps (→~17 after strength 0.7), g=2.5, seed 0, RTX A5000. PIE-Bench is the substrate — the only repo subset with an explicit source_prompt and target_prompt per example, exactly what Δ_edit needs. All 24 examples (8 task types × 3). A custom denoising loop evaluates the transformer twice per step at the same x_t (source / target conditioning) to read v_src, v_edit, Δ_edit.

Diagnostic framing. Skip schedules are oracle-derived from the reference trajectory's own signal stability: for a target skip ratio ρ, reuse the ρ-fraction of interior steps where that variant's change signal is smallest (endpoints never skipped). Every variant skips at the same ρ, isolating "is this signal a better guide to where reuse is safe?" The cacheability metric is absolute adjacent L2 — the velocity error a stale reuse injects into the stepping update (v_src is recomputed exactly for delta caching, so both signals are in the same units).

Delta caching cuts reconstruction error ~13× at equal skip. On the Pareto subset (8 examples × 5 operating points) delta caches hold DINO ≈0.985 / LPIPS ≤0.02 out to 80% skip while full caches collapse to DINO ≈0.83 / LPIPS ≈0.18 — delta dominates the whole frontier; spectral helps within each family.

Verdict

STRONG GO. Caching the edit direction is both measurably smoother (2.4×, 96% of steps) and yields a strictly better speed–quality frontier than the SeaCache-style full-prediction baseline, across all operating points and categories. Honest accounting: at equal skip ratio delta caching does more forwards (it keeps the base v_src live), so its win is realised under true-CFG editing where both branches run anyway; we report quality-vs-skip as the primary axis and speedups under both naive and true-CFG models.

Limitations

  • 24 examples (PIE-Bench is the only paired-prompt subset); Pareto sweep on an 8-example balanced subset.
  • Oracle skip schedules isolate the signal question; a deployed cache needs an online rule (extrapolated low-pass change) — future work.
  • SDEdit edits are weak per CLIP-T (gain ≈0 incl. the reference); the cache comparison is clean regardless, since every variant is scored against the same reference edit.
  • img2img, not Kontext — chosen because it exposes a clean source/target prompt pair at 512px.

Next (video follow-up)

Port the Δ_edit low-pass cache to a video-editing diffusion path (per-frame source/target or shared instruction), add an online skip rule and verify it recovers most of the oracle frontier, and measure temporal-consistency-vs-speedup against per-frame SeaCache. Target ≥1.7× true-CFG speedup at ≤0.02 LPIPS regression on 30–50 clips.
